## ilink-hub Logo 设计

- 深度理解了项目：ilink-hub 是 Rust 实现的透明 iLink 代理/多路复用 Hub——Hub 作为唯一连接持有者收口真实微信 iLink 连接，再扇出给多个 AI 后端（Claude Code / Cursor / Recursive / Codex / OpenClaw 等），客户端零改造。Bridge = 客户端适配层。
- 现有品牌（`docs/public/logo.svg`、`favicon.svg`）：紫蓝渐变 `#6366F1→#8B5CF6` 圆盘 + 中心 hub + 6 条辐条到卫星节点。hub-and-spoke 几何本身很贴切（多路复用），但配色是通用 AI 紫，没讲出产品故事。
- 新方案（已产出 SVG 文件）：保留 hub-and-spoke 几何；配色改为 **绿→橙渐变 `#12B76A → #F97316`**，象征在微信 iLink（绿）与 AI/Rust 内核（橙）之间桥接；中心双箭头 `>>` 代表透明转发/代理。另提供单色版（墨黑 `#0E1A17`）。
- 交付文件：`ilink-hub-mark-color.svg`、`ilink-hub-mark-mono.svg`、`ilink-hub-favicon.svg`（项目根目录）。
- Ardot 画布（文件 `704414264706579` "iLink-Hub-Logo"）部分建成：Header + 主标识 section + 可复用 color master 组件（id `2:70`）。构建 Lockups section 时 Ardot adapter 持续 `NO_ADAPTER`，完整品牌板（组合/调色板/单色反白/缩放）未完成。adapter 恢复后从 `2:1` 继续，引用 `2:70` 即可。
- 备注：batch_edit 大 payload（含 2 个内嵌 SVG）会触发参数校验 "root: must be object" 的偶发解析失败；规避办法是把 mark 做成可复用组件后用 ref 引用，单批只放一个内嵌 SVG。
