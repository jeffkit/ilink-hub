# 微信命令

在微信中向 AI 助手发送以下命令，可以控制 iLink Hub 的路由行为。

::: tip
这些命令由 Hub 拦截处理，**不会**传递给 AI 后端。
:::

## 命令列表

### `/list` — 查看所有客户端

显示所有已注册的后端及其在线状态：

```
/list
```

**回复示例：**

```
已注册客户端：
  ● mac-home（Mac 本机）—— 在线，最后活跃：2 秒前
  ● server-prod（生产服务器）—— 在线，最后活跃：8 秒前
  ○ openclaw-01（OpenClaw 实例 1）—— 离线，最后活跃：5 分钟前
  
当前活跃：mac-home
```

### `/use <name>` — 切换活跃后端

将后续所有消息路由到指定客户端：

```
/use server-prod
```

::: tip Bridge 也是普通客户端
[`ilink-hub-bridge`](/bridge/quick-try) 注册时同样有一个 `--name`，例如 `local-bridge`。只有执行 `/use local-bridge` 后，微信里的普通消息才会交给该进程上的本地 CLI。
:::

**回复示例：**

```
✓ 已切换到：server-prod（生产服务器）
```

如果指定的客户端不存在或离线，Hub 会提示并保持当前路由不变。

### `@<name> <消息>` — 临时 @ 一个后端（新建会话）

无需先 `/use` 切换后端，直接 `@` 一个后端并发送消息，即可**临时**在该后端上**新建一个会话**处理这条消息。这是一种快捷方式，性质与「引用回复」类似：

- **不改变**你当前 `/use` 的后端，也**不改变**任何后端的活跃会话。
- 每次 `@` 都会创建一个全新的会话（名称形如 `at-20260614-203012123`）。
- 这里的 `<name>` 与 `/use <name>` 使用的名称完全一致（可用 `/list` 查看）。
- 后端名取**第一个空格之前**的部分，其余内容为消息正文。

```
@server-prod 帮我看下生产日志最近的报错
```

::: tip 如何继续这个临时会话
因为 `@` 不会切换当前后端/会话，要在这个临时会话上继续对话，**引用**该后端的回复再发送即可——Hub 会把后续消息精确路由回这个临时会话。
:::

如果 `@` 后面的名称不是已注册的后端，整条消息会被当作普通文本，按当前路由正常发送（不会剥离 `@` 前缀）。

### `/session …` — 管理同一后端下的多会话

在当前后端下管理多个并行会话：

```
/session list              # 列出当前对话的所有 session
/session new <名称> [UUID]  # 新建 session（可选初始 UUID）
/session use <名称>         # 切换活跃 session
/session delete <名称>      # 删除 session
```

### 引用回复 — 临时回到某条消息的来源

引用某条机器人消息后再发送内容，会**优先**路由到发出该条消息的后端及其会话（或重新执行对应的 Hub 指令），无需依赖当前 `/use`。`@<name>` 与引用回复都属于此类「临时操作」。

### `/broadcast <消息>` — 向所有在线后端广播

将消息同时发送给所有在线的客户端：

```
/broadcast 今天有紧急任务，请所有 AI 助手注意
```

**回复示例：**

```
✓ 已广播到 2 个在线客户端：
  - mac-home（Mac 本机）
  - server-prod（生产服务器）
```

### `/status` — 查看 Hub 状态

显示 Hub 的整体运行状态：

```
/status
```

**回复示例：**

```
iLink Hub v0.1.4 状态：
  上游连接：已连接
  在线客户端：2 / 3
  消息队列：mac-home(0), server-prod(2), openclaw-01(-)
  运行时间：3 天 2 小时 15 分钟
```

## 命令不响应怎么办？

1. 确认 Hub 服务正在运行（`/health` 端点返回 200）
2. 检查 Hub 日志是否有错误
3. 确认当前账号有已注册的客户端（`/list` 应该返回列表）

## 自定义回复

当所有客户端都离线时，Hub 会向微信用户发送友好的提示信息，告知当前无可用 AI 后端。这个回复内容可在未来版本中自定义配置。
