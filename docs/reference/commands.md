# 微信命令

在微信中向 AI 助手发送以下命令，可以控制 iLink Hub 的路由行为。

::: tip
这些命令由 Hub 拦截处理，**不会**传递给 AI 后端。
:::

## 命令列表

### `/list` — 查看所有客户端

显示所有已注册的后端及其在线状态：

```
/list
```

**回复示例：**

```
已注册客户端：
  ● mac-home（Mac 本机）—— 在线，最后活跃：2 秒前
  ● server-prod（生产服务器）—— 在线，最后活跃：8 秒前
  ○ openclaw-01（OpenClaw 实例 1）—— 离线，最后活跃：5 分钟前
  
当前活跃：mac-home
```

### `/use <name>` — 切换活跃后端

将后续所有消息路由到指定客户端：

```
/use server-prod
```

**回复示例：**

```
✓ 已切换到：server-prod（生产服务器）
```

如果指定的客户端不存在或离线，Hub 会提示并保持当前路由不变。

### `/broadcast <消息>` — 向所有在线后端广播

将消息同时发送给所有在线的客户端：

```
/broadcast 今天有紧急任务，请所有 AI 助手注意
```

**回复示例：**

```
✓ 已广播到 2 个在线客户端：
  - mac-home（Mac 本机）
  - server-prod（生产服务器）
```

### `/status` — 查看 Hub 状态

显示 Hub 的整体运行状态：

```
/status
```

**回复示例：**

```
iLink Hub v0.1.4 状态：
  上游连接：已连接
  在线客户端：2 / 3
  消息队列：mac-home(0), server-prod(2), openclaw-01(-)
  运行时间：3 天 2 小时 15 分钟
```

## 命令不响应怎么办？

1. 确认 Hub 服务正在运行（`/health` 端点返回 200）
2. 检查 Hub 日志是否有错误
3. 确认当前账号有已注册的客户端（`/list` 应该返回列表）

## 自定义回复

当所有客户端都离线时，Hub 会向微信用户发送友好的提示信息，告知当前无可用 AI 后端。这个回复内容可在未来版本中自定义配置。
